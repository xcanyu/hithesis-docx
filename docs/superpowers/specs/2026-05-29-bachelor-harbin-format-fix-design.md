## 3. 技术方案

### 3.1 备份与回退策略

#### Git 分支策略
```bash
# 1. 提交当前状态作为备份
git add -A
git commit -m "backup: 格式修复前的状态备份"

# 2. 创建修复分支
git checkout -b fix/bachelor-harbin-format

# 3. 每完成一个任务单独提交
git add hitthesis/config.py
git commit -m "fix: 修复页面边距 (P0)"

# 4. 不满意时回退单个任务
git revert HEAD  # 回退最近一次提交
# 或
git reset --hard HEAD~1  # 丢弃最近一次提交
```

#### 备份文件策略
- **修改前**：保留原始文件副本（可选）
- **修改后**：通过 git 管理版本
- **回退方式**：git revert 或 git reset

#### 并行修改策略
使用 subagent 并行处理独立任务时：
- 每个 subagent 修改不同文件
- 完成后合并到主分支
- 冲突时手动解决

### 3.2 修复策略
- **渐进式修复**：每个模块独立修改
- **可回退性**：每完成一个模块，用户验证后再继续
- **并行处理**：使用 subagent 处理独立任务

### 3.2 文件修改计划

#### 配置文件
- `hitthesis/config.py` - 更新所有格式常量

#### 核心模块
- `hitthesis/document.py` - 修改页眉页脚、章节标题
- `hitthesis/cover.py` - 修改封面间距

#### 辅助模块
- `hitthesis/authorization.py` - 验证授权声明格式
- `hitthesis/compile.py` - 验证编译流程
