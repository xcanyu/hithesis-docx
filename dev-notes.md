### 痛点：cover1 的 P1、P2 无法用 VBA 精确测量

VBA 的 `wdVerticalPositionRelativeToPage` 测量的是段落顶部绝对位置。cover1 的 P1（"本科毕业论文"）和 P2（论文标题）在参考模板中有注释行（`↑`、`（格式说明）`）占据垂直空间，导致：

- VBA 测得的参考模板 P1→P2 gap 包含注释行空间（78.0pt）
- 生成文档没有注释行，P1→P2 gap 为纯段落间距（94.3pt）
- 两者差 16.3pt（≈5.7mm），但直尺实测仅 1-2mm 误差

**结论**：VBA 绝对位置测量受注释行干扰，直尺实测更可靠。

## 子图功能（2026-06-07）

新增 `add_subfigure()` 方法，支持并排和网格布局。

### API

```python
doc.add_subfigure(
    subfigures=[(image_path, subcaption), ...],
    caption="总图题",
    ref="label",
    cols=None  # 每行列数，默认并排
)
```

### 布局示例

| 布局 | 用法 | 示例 |
|------|------|------|
| 并排 | `cols=None`（默认） | 2图、3图并排 |
| 2x2 网格 | `cols=2` | 4图分两行 |
| 3列网格 | `cols=3` | 6图分两行 |

### 实现方式

- 用表格实现并排/网格布局
- 自动清除表格边框
- 每个子图有 (a)(b)(c) 标号
- 总图题在表格下方居中
- 支持交叉引用 `[cite:label]`

### 经验总结

1. **VBA gap 测量**：参考模板有注释行时，content-to-content gap 包含注释行空间，不能直接用绝对位置反推参数
2. **直尺验证**：封面精度要求高时，直尺实测比 VBA 更可靠（1-2mm 误差可接受）
3. **page_break_before 黑点**：Word 对 page_break_before 段落会显示黑点（类似标题标记），改用分节符可解决
4. **空格定位**：用 Times New Roman 空格推算位置时，需实测校准（每个 TNR 空格约 3pt）
